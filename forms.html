Checkboxes

A label variant class of label--switch is used for side-by-side inputs and labels.

For radios and checkboxes, we put the input element inside the label element.

A class of checkbox is required on checkbox input elements.

<label for="check1" class="label--switch chunk">
    <input id="check1" class="checkbox" value="green" type="checkbox" />
    Green
</label>
<label for="check2" class="label--switch chunk">
    <input id="check2" class="checkbox" value="blue" type="checkbox" />
    Blue
</label>

Validation errors

Error styles are applied to any child input of an element with the class hasError.

Error text is set explicitly with text--error.

First name can't be empty

<div class="chunk hasError">
    <label for="test-textinput">First name</label>
    <input id="test-textinput" type="text" value="" />
    <p class="text--error">First name can't be empty</p>
</div>

Launcher

Applies form field styles to an anchor, for use in launching custom input (ie. date/location pickers).

Boise, Idaho 
<div class="chunk">
    <label for="specialPicker2">Pick a location</label>
    <a id="specialPicker2" class="launcher" href="#">Boise, Idaho</a>
</div>

Launcher variants

A class of launcher is required for launcher anchors. The following classes are optional variants

Class Description
launcher--placeholder Applies placeholder-like text styles for "unselected" values
